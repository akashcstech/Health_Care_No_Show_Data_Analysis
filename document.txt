*Data Exploration and Tranformation*

1.Age values equal to 0 were treated as invalid because records existed with age = 0 
and adult characteristics (e.g., divorced marital status). These values were converted 
to missing and imputed using the median age.

2.Age values which are less than 18 having adult characteristics (e.g., divorced marital status).
These Values contain inconsistencies. Data quality checks revealed inconsistent records, 
such as patients aged 9 marked as retired or married. These records were treated as data quality issues

Data quality assessment identified unrealistic combinations (e.g., patients under 18 marked as married, 
retired, or having children), suggesting that some features may be synthetically generated or contain inconsistencies

3.Missing values OF 1500 in categorical variables such as education_level were imputed using the
category "Unknown" to preserve records while explicitly indicating unavailable information.
